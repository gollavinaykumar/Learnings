Most developers know these commands:

```bash
cat /proc/cpuinfo

cat /proc/meminfo

ls /sys
```

They use them every day.

But very few understand:

- Why do these files exist?
- Where are they stored?
- Why don't they consume disk space?
- Why does changing a value in `/proc` or `/sys` immediately affect the kernel?
- Why does Linux expose its internal data as files?

The answer is one of Linux's greatest design philosophies:

> **Everything is a file.**

The Linux kernel exposes itself through **Virtual File Systems (VFS)**, allowing users and applications to inspect and even configure the running kernel using ordinary file operations.

After this chapter, you'll understand how Linux exposes its internals through `/proc` and `/sys`, and why these directories are essential for system administration, debugging, Docker, Kubernetes, and performance tuning.

# 📖 Everything Is a File

Linux has a unique philosophy.

Instead of creating special APIs for everything,

Linux exposes resources as files.

Examples:

```
Regular File

↓

/home/user/file.txt
```

```
Disk

↓

/dev/sda
```

```
Terminal

↓

/dev/tty
```

```
Running Process

↓

/proc/1234
```

```
Kernel Parameters

↓

/proc/sys
```

```
Hardware Devices

↓

/sys
```

Everything becomes accessible through familiar file operations.

---

# What is a Virtual File System?

Suppose you open:

```bash
cat /proc/meminfo
```

Where is that file stored?

Not on the SSD.

Instead,

the kernel generates it **on demand**.

```
Application

↓

Kernel

↓

Generate Content

↓

Display
```

No disk access occurs.

---

# Virtual File Systems

Unlike ext4 or XFS,

Virtual File Systems don't store data permanently.

```
Virtual File

↓

Generated by Kernel

↓

Returned to User
```

Examples:

```
procfs

sysfs

tmpfs
```

---

# Why Does Linux Use Virtual Filesystems?

Without them,

Linux would need hundreds of special commands.

Instead,

everything becomes:

```
open()

↓

read()

↓

write()

↓

close()
```

Applications don't need to learn new APIs.

---

# What is `/proc`?

`/proc` is a virtual filesystem that exposes:

- Running processes
- Memory information
- CPU information
- Kernel information
- Network information

Everything is generated dynamically.

# What is `sysctl`?

Instead of editing files directly,

Linux provides:

```bash
sysctl
```

Example:

```bash
sysctl net.ipv4.ip_forward
```

Change:

```bash
sudo sysctl -w net.ipv4.ip_forward=1
```

The kernel updates immediately.

---

# Why `sysctl` Matters

Administrators tune:

- Networking
- Memory
- Security
- Virtual Memory

without rebooting.

# `/sys` vs `/proc`

| `/proc` | `/sys` |
|----------|---------|
| Process Information | Device Information |
| CPU & Memory | Hardware Hierarchy |
| Kernel Runtime Data | Kernel Objects |
| Process Directories | Device Directories |

# Interview Questions

## What is `/proc`?

`/proc` is a virtual filesystem that exposes runtime information about processes, memory, CPU, networking, and the Linux kernel.

---

## What is `/sys`?

`/sys` is a virtual filesystem that exposes kernel objects, hardware devices, and driver information.

---

## Are `/proc` files stored on disk?

No.

The kernel generates their contents dynamically whenever they are read.

---

## What is `sysctl` used for?

`sysctl` reads and modifies kernel parameters at runtime.

---

## What is the main difference between `/proc` and `/sys`?

`/proc` primarily provides runtime process and kernel information.

`/sys` primarily represents hardware devices and kernel objects.

---

## Why does Linux expose kernel information as files?

Because Linux follows the philosophy of providing a simple, consistent interface through standard file operations.

---

# Summary

Linux exposes its internal state through virtual filesystems.

```
Application

↓

read()

↓

Virtual File

↓

Kernel

↓

Generate Information

↓

Application
```

Key concepts:

- `/proc` and `/sys` are virtual filesystems.
- Their contents are generated dynamically by the kernel.
- `/proc` focuses on processes and runtime system information.
- `/sys` focuses on devices, drivers, and kernel objects.
- `sysctl` allows runtime kernel tuning.
- Most Linux monitoring, debugging, and container tools depend on these filesystems.

Understanding `/proc` and `/sys` is essential before learning namespaces, cgroups, networking, Docker internals, and Kubernetes because these technologies rely heavily on information exposed by the Linux kernel.
